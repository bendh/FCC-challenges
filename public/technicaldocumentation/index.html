Java and Kotlin

Java was the first language to run on the JVM. JVM stands for Java virtual machine. The JVM provides a unified runtime to run Java code on any platform. The credo of Java is "write once, run anywhere".

Kotlin is a new language for the JVM designed by Jetbrains. It has taken the best features of Java, C# and Javascript. It's a fun and productive language to use.

Kotlin is the prefered language for writing Android apps.

Kotlin is expanding beyon the JVM and targeting also native platforms.

Hello world

In this section we will write a Hello world console program in Java and Kotlin.

Every Java runnable program needs to have a class with a public static main method. To print to the console you need the System.out.print function.

public class HelloWorld { public static void main(String[] args) { System.out.println("Hello world"); } }

For Kotlin you will need the same but the code is less verbose as you can see

fun main(args: Array) { println("Hello, world!") } There are some diference to note:
  • The kotlin code don't need to be in a class
  • Because the main function is written in a package file, you can omit the method modifiers public and static
  • No comma's are needed in kotlin
  • println is called directly
  • type definition is placed after the variable name
Objects and primitives

In this section we will discuss the difference between Object's and primitive types

To declare a variable in Java you specify the access modifier, type and name. To declare a private String called name you would write: private String name;

This name variable is declared but not yet initialised. If you try to access this variable you will be a dreaded Nullpointer exception

private String name; int lenght = name.lenght(); // Will throw NullPointerException ;<(

Primitives like boolean, int, long and double are always initialised

int age; //Initialised with the value 0;
